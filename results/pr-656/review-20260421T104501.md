# Executive Summary

本 PR 主要解决了 **Issue #411** 中提出的「当所有 Worker 都不可用时，客户端应能够实现 Fail-Fast（快速失败）」的需求。通过引入 `WorkerSwitchState` 状态机、`switchGeneration_` 周期管理以及 `SwitchTriggerReason` 触发原因区分，增强了客户端在 Worker 切换过程中的健壮性和并发安全性。

**核心改进：**
1. **状态机管理**：新增 `AVAILABLE`、`SWITCHING`、`NO_SWITCHABLE_WORKER` 三种状态，明确了切换中的中间态和最终不可用态。
2. **Fail-Fast 机制**：在 `GetAvailableWorkerApi` 中增加了对 `NO_SWITCHABLE_WORKER` 状态的检查，避免了在明知无可用 Worker 时仍进行无效的 RPC 尝试。
3. **异步安全性**：引入 `switchGeneration_`（版本号/代数）机制，有效防止了旧的异步切换回调在切换完成后错误地覆盖当前状态（ABA 问题）。
4. **逻辑重构**：将复杂的 `SwitchWorkerNode` 拆分为多个小函数，提升了代码的可读性和可测试性。

**风险评估**：
- **P1**：异步切换状态机的复杂性较高，需确保在所有异常路径（如超时、初始化失败）下都能正确恢复或进入错误态。经审查，PR 涵盖了主要的 recovery 路径。
- **P2**：对于 `LOCAL_WORKER` 的主动缩容（Voluntary Scale Down），若无备选节点，目前逻辑选择「保持当前节点」而非立即 Fail-Fast。这在某些场景下可能是合理的最佳尝试，但需确认是否符合所有业务预期。

## A. 功能与需求符合性

- **符合性 (Issue #411)**：PR 完美实现了 Issue 要求。通过 `MarkNoSwitchableWorkerLocked` 标记不可用状态，并在业务入口处拦截请求。
- **差异化处理**：区分了 `WORKER_UNAVAILABLE`（非预期故障）和 `VOLUNTARY_SCALE_DOWN`（主动缩容）。在主动缩容场景下，增加了 `keepCurrentWorker` 的回退逻辑，体现了设计上的完备性。
- **测试覆盖**：新增的 ST 测试用例模拟了全集群 Worker 依次缩容的场景，验证了客户端最终进入 Fail-Fast 状态并返回错误，闭环了需求。

## B. 性能与可扩展性

- **锁粒度优化**：`TrySwitchToStandbyWorker` 将耗时的 `Init()` 和 `StartListenWorker()`（含网络连接）置于 `switchNodeMutex_` 锁范围外，仅在最终提交状态时通过 `CommitStandbySwitch` 持锁，避免了切换期间阻塞其他非切换相关的元数据读取。
- **代数管理**：`switchGeneration_` 的使用避免了复杂的锁竞争，通过简单的版本比对实现了并发控制，性能开销极低。

## C. 潜在缺陷（Bug / 竞态 / 资源）

### P1 级问题：无

### P2 级问题

- **位置**：`src/datasystem/client/object_cache/object_client_impl.cpp`:700
- **摘录**：
  ```cpp
  +    const bool keepCurrentWorker =
  +        reason == client::SwitchTriggerReason::VOLUNTARY_SCALE_DOWN && current == LOCAL_WORKER;
  ```
- **问题**：如果 `LOCAL_WORKER` 确实要强制下线（主动缩容），且此时没有备选 Standby。逻辑会选择 `RestoreWorkerAvailableIfNeeded` 恢复 `AVAILABLE` 状态。虽然这允许了「最后的尝试」，但也可能导致客户端在 Worker 已经关闭后仍持续尝试连接，直到触发下一次心跳失败。
- **建议**：建议在日志中明确记录「No standby available during voluntary scale down, staying on current worker」，以便于排查缩容时的长尾延迟问题。

### P3 级问题

- **位置**：`src/datasystem/client/listen_worker.cpp`:410
- **摘录**：
  ```cpp
  +        isSwitched_ = !switchWorkerHandle_(index_, SwitchTriggerReason::WORKER_UNAVAILABLE);
  ```
- **问题**：在 `TrySwitchBackToLocalWorker`（从备选切回本地）时，固定使用了 `WORKER_UNAVAILABLE`。虽然目前 `SwitchWorkerNode` 对 `LOCAL_WORKER` 忽略了 reason，但语义上略有不妥。
- **建议**：未来若 reason 用于度量，建议增加 `RECOVER_LOCAL` 枚举项。

## E. 具体优化与重构（可执行清单）

1. **[E-1] [P2] 维度：功能**
   - **位置**：`src/datasystem/client/object_cache/object_client_impl.cpp`:1160
   - **摘录**：
     ```cpp
     +    if (workerSwitchState_ == WorkerSwitchState::NO_SWITCHABLE_WORKER) {
     +        return NoSwitchableWorkerStatus();

     ```
   - **问题**：在 `NO_SWITCHABLE_WORKER` 状态下，所有的 API 调用都会被拦截。
